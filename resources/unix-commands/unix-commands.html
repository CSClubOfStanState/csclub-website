Unix shortcuts and wildcards

Some shortcuts and wildcards commonly used in Unix are listed here.

  • / (forward slash) The forward slash alone denotes the root of the file system tree
  • ~ (tilde) stands for the your home directory
  • . (single period) stands for the current directory
  • .. (double period) refers to the parent directory of the current directory (the directory one level closer to the root of the file system)
  • * (asterisk) stands for zero or more of any character
  • ? (question mark) stands for any single character in a file
  • | (vertical bar) activates a Unix "pipe" from one command to the next command
    • wq

pwd, ls, mkdir, cd, and cp

The following are some commonly used commands.

The pwd command (or print working directory) is used to output the path of the current working directory.

The ls command lists the content of a directory.

The mkdir command (or make directory) is used to create a new directory.

The cd command (or change directory) changes the current working directory.

The cp command is used to copy files and directories.

The cat command

cat prints the contents of files passed to it as arguments and can be used to combine, and create new files. cat is frquently seen in unix command pipelines.

The following example prints the cocatenated contents of file1 followed by file2 to stdout cat file1 file2

The following example prints file with line numbers cat -n file.txt

The folllowing example prints the count of the number of lines in a file cat file.txt | wc -l

zcat allows you to cat zipped files

The following example prints the contents of a file in reverse order, bottom first, with tac (or cat spelled backwards).

The grep command

The grep command is the terminal's analog of find from ordinary computing (not to be confused with unix find). The grep command searches the given file for lines containing a match to the given "strings" or words.

The following example returns lines with the text apple grep apple myfile.txt

The following examples returns numbered lines grep -n apple myfile.txt

The following example returns lines with case insensitive matching grep -i apple myfile.txt

The following example returns colored lines the matching text grep --color apple myfile.txt

Also useful are what are somtimes called the ABCs of Grep—that's After, Before, Context.

The following example returns lines with the match as well as 1 After grep -A1 apple myfile.txt

The following example returns lines with the match as well as 2 Before grep -B1 apple myfile.txt

The following example returns lines with the match as well as 3 After and Before grep -C3 apple myfile.txt

Redirecting output using " > "

The symbol " > " is typically use to redirect the output of a command (generally) into a file.

The following example will will write the text "Have a good day" into a file called "newFile.txt" echo "Have a good day" > newFile.txt

The folling example will write the output from a grep command and write the lines into a new file. grep apple fruits.txt > newFile.txt

Finding what you need using the find command

The find command recursively descends the directory tree for each path listed, evaluating an expression (composed of the “primaries” and “operands” listed below) in terms of each file in the tree. (Many more search criteria can be used. Use "man find" to learn more.)

The following example returns the file names in our subfolders for a file named "findme.txt" find . -name findme.txt

The following example returns the file names in our subfolders for a file named findme with any file extensions, ignoring the character's case sensitivity. find . -iname findme*

The following example returns the file names in our subfolder "projects" for a file named smallproject with any file extensions, ignoring the character's case sensitivity. find projects -iname smallproject*

Redirecting the output using " | "

The symbol " | " is use to send the output of a command to the input of another. This process is know as piping.

The folllowing example pipes our output from ls into a grep command. ls | grep apple

The folllowing example pipes our output from ls into a grep command and pipes that output into a word count command. ls | grep apple | wc -l

Looking at your history

You have been working for a while, and want to re-examine your previous actions.

  • Type in the terminal command line: history .
  • On the list produced, find one of the earlier grep commands you ran. Each line of your list is numbered on the left. Suppose the number on the leftmost end of the line of the grep command is 5. Then, to run that grep command again, you type at the terminal command line: !5
  • Use history to run one of your previous grep commands

Looking at processes using ps & sleep command

The ps command can be used to examine processes running on your Unix system.

The following example shows process info ps

The following example shows verbose process info ps -f

The following example shows verbose process info for all processes running on the machineps -f -A

The command sleep pauses for an amount of time specified in seconds that can use an ampersand to run things in the background.

The following example creates a sleep process for 15 seconds and hands the terminal to the foreground. sleep 15 &

The following example creates two sleep processes for 500 and 600 seconds and hands the terminal to the foreground. sleep 500 & sleep 600 &

The following example finds all processes runnning on the machine and | (pipes) that output to the grep command that searches for the keyword sleep that | (pipes) that output to the wc (word count) command that counts the number of lines returned by the previous output. ps -A | grep sleep | wc -l
