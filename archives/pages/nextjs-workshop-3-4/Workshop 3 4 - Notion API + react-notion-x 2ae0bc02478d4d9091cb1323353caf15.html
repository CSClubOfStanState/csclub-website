Description 🗨️

This workshop aims to teach you how to use an open-source library for your personal use and to deploy a new NextJS Project with your new blog powered by the Notion API.

Requirements 📃

This workshop supplements the previous workshop where we created a NextJS web app and deployed our portfolio site onto Vercel. If you have not completed the previous workshop, feel free to clone my repo and use it as the basis for this workshop.

There is a TIPS and FAQs Section at the bottom; use it if you get stuck; otherwise, just ask!!!

Want to Accomplish 🤔

  • Learn a bit about open-source
  • Learn to read the documentation from a library
  • Learn to use open source code for personal projects
  • Dive right into using the Notion API
  • Recreate our portfolio with an addition of a blog using the Notion API
  • Deploy our portfolio with our blog onto Vercel

Set Up 🔨

Clone the following repository into a local directory: https://github.com/NotionX/react-notion-x and launch the repository in VSCode.

git clone git@github.com:NotionX/react-notion-x.git

Run the following commands inside of react-notion-x/examples/minimal

npm install --global yarn
yarn upgrade next@latest
yarn upgrade react@latest react-dom@latest
yarn install
npm install
yarn dev
Tread carefully

You should now see the following:

We aim to adopt the source code from this repository into our repo. The reason is that our website had a section for a blog. The Notion API will serve as our Content Management System for our blog.

We want to change the current Notion Page that it is observing to our workspace. Follow the steps in #getting-started to get our app's necessary ids and credentials. This should’ve been completed in the onboarding for notion page.

Going Private (Optional)

We can use a private Notion workspace if we do not want the world to see our private pages in that workspace. To do so, there are extra steps involved.

For accessing private pages, make sure to give your client access to the authToken and activeUser. Both can be retrieved from within your site, as long as it is open.

Open your dev tools > Application > Cookie > and copy/paste the token_v2 AND notion_user_id. Respectively, activeUser: notion_user_id, authToken: token_v2.

Here was my code snippet for the client usage:

const notion = new NotionAPI({
  activeUser: process.env.NOTION_ACTIVE_USER, 
  authToken: process.env.COOKIE
  })

Dot Env

As you can see above, we are referencing something like process.env.NOTION_ACTIVE_USER . We are calling for a key-value pair from a .env file. Create a .env file at the root of your directory. Make sure that your .gitignore has .env in it! We do this to make sure that our secrets remain a secret. Restart the local host instance and you should see an info prompt at the bottom from NextJS.

info - Loaded env from /Users/UserName/Directory/workshops/minimal/.env

With all your credentials and pageIds changed, launch your private page! (If you chose to do the optional stage...)

Tips/FAQs 🤔

  1. If you are on mac and cannot view some files on your finder because they are hidden, hit the following strokes in succession → ⌘ + ⇧ + .
  1. If you run into errors when trying to run npm run dev make sure to have the expected nodejs version. If you are unsure of it, check the documentation for the Github repository and NextJS.
  1. 🗒️ Note: There is a 🐞bug that appears where the page Id cannot be found. In this situation, restart the local host.
  1. Q: Can I run yarn run instead of npm run dev? A: Yes, but just know that we are using npm because it is easier to debug. 🐛
  1. Q: What if I get a pageId error? A: It depends on what step you are at; there’s a chance your URLs at some point are a bit messed up.
  1. Q: I see the error: Cannot find module, @next/bundle-analyzer , what do I do? A: Install the bundler: npm install @next/bundle-analyzer
  1. Q: What do I do if add yarn types issue arose during deployment? A: Make sure to install Yarn using npm: npm install --global yarn. Push your changes again
  1. Q: Do I need to do the optional step? A: No, if you do not want to, you do not have to, but to demonstrate authorization for APIs, I will cover this.
  1. Q: Do I need to be familiar with Notion and have an account to attend this workshop? A: Yes and No. Check out Notion real quick; make an account if you’d like. If you are an AVID user, then this workshop is for you. ❤️
  1. If you get stuck, ask questions!!!!
