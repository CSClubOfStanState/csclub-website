What's in Our Project ‚ÅâÔ∏è

For the sake of time, I will briefly go over each file, its importance in the base app, and what we will be doing with it.

_app.js - The main file that renders the whole page. Any components declared here get reused across the whole page.

index.js - The backbone of our page that we can manipulate to render the root page ("https://PageName").

globals.css - The cascading style sheet that never sees the light of day except serves as the backup for our fonts and look of the original page, which we can use for dark mode.

hello.js - If we were making APIs and would want people to fetch data from our application, they could hit this page‚Äôs endpoint like so: https://site/hello/ ‚Üí returns data

/public/ - this folder hosts a lot of our images or any files that we would want the end-user to have downloaded on their site upon rendering a page.

Yes! This is the SAME CSS that we used for the previous page. The catch is that Home.module.css is the same file as index.css from workshop 1, we are merely reusing the code that we know works. You might notice that the naming scheme is different. The reason for this is that NextJS does a great job of organizing CSS at the component level. This means that every single page and every small new component should have its own CSS file to ensure that we do not have issues with CSS leaking on top of each other. (It also helps to keep things clean, which is always a plus the more complex your project becomes)

Q: How do we reference our CSS style sheet?

A: We simply import it! If you look at the top of your index.js page you will see the single line of code that imports our sheet: import styles from "../styles/Home.module.css";
