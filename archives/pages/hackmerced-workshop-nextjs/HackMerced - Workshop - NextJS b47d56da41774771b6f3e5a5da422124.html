What's in Our Project ⁉️

For the sake of time, I will briefly go over each file and its importance in the base app and what we will be doing with it.

_app.js - The main file that renders the whole page. Any components declared here get reused across the whole page.

index.js - The backbone of our page that we can manipulate to render the root page ("https://PageName").

globals.css - The cascading style sheet that never sees the light of day, except, serves as the backup for our fonts and look of the original page which we can use for dark mode.

hello.js - If we were making APIs and would want people to fetch data from our application, they could hit this page’s endpoint like so: https://site/hello/ → returns data

/public/ - this folder hosts a lot of our images or any files that we would want the end-user to have downloaded on their site upon rendering a page.

Q: How do we reference our CSS style sheet?

A: We simply import it! If you look at the top of your index.js page you will see the single line of code that imports our sheet: import styles from "../styles/Home.module.css";

🚨 Just so you know, all ampersands need to be converted to their ASCII equivalents. E.g., &38; and single apostrophe requires &39;

Before we decide to deploy our app, let’s create reference links to the following pages: Blog, About, and Projects. One of the many great things about NextJS is that the Routing is simplified by file structure. This means that if we create a file named about.js and place it inside of the pages/ directory, then we have successfully wired up the route for the about page. We can reference the following documentation from NextJS to create a new page. Pages in NextJS

Example: If you create pages/about.js that exports a React component like below, it will be accessible at /about.

🔔Remember, to manually change all the href links to point to /about href="/about" For us, this is in our Navigation. Below is our javascript code for our about page.

function About() {
  return <h1>About</h1>}

export default About

We can follow suit with other pages. If you have done everything correctly, you can then use any navigation link to traverse to a new page.

We can clean up our links from being <a href="google.com" /> and place each of them inside a nextjs/link like so: 

<nav className="nav-container">
		<Link href="/blog" passHref>
			<a className={styles.nav_links}>Blog</a>
		</Link>

		<Link href="/about" passHref>
			<a className="nav-links">About</a>
		</Link>

		<Link href="/projects" passHref>
			<a className="nav-links">Projects</a>
		</Link>

		<Link href="https://github.com/ggiande" passHref>
			<a className="nav-links">Github</a>
		</Link>
</nav>

If you notice an issue with <Link /> not being recognized, make sure to import the module import Link from "next/link"; at the very top of index.js
