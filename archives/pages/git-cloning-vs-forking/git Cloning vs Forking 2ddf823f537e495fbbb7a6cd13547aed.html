git Cloning vs Forking

Assuming we know what git is and what Github is, we can discuss what cloning vs forking a project is and is not.

Cloning

Cloning is what it sounds like! We are merely making a copy for ourselves and keeping it as our own.

  • We are not saying who the owner is
  • We are taking the current version and are not expecting updates from the previous owner.
  • This is now ours, so we can play with it however we please
  • Our clone can exist on GitHub as our own project
  • Our clone can exist on our own machine as our own local project.

An example of cloning is the nextjs-notion-starter-kit by transitive-bullshit

Forking

Forking is another way of saying:

I am taking your project, choosing to make my own version of it, and taking any updates you make and maybe making them mine too
  • Can choose to support the original project with my source code by contributing
  • Can raise a pull request to merge my project with the owners
  • Everyone will know that I based my project on yours for eternity

An example of forking is DarkAudacity for Windows
