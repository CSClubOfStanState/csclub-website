Setup and Installations

Before we can start modding, we’ll need to set up a few things so we can mod in the correct environment. I’ve included some explanation on what each of these things is and how to install them below.

These are: JDK 17, IntelliJ, WinRAR (or equivalent), Fabric Example Mod Template

JDK

This is the Java Development Kit, a set of tools and classes that Oracle provides for developers such as us we will use to make our mods. This allows us to do all sorts of cool tricks like creating and modifying registries, building mixins, calling on and modifying superclasses, and more. All of the aforementioned and more will be future activities in these workshops. Hopefully, in the end, you will understand what these concepts are and how to execute them.

Previous versions of Minecraft use older versions of the JDK, and while most are backward-compatible (as in you can use a newer JDK to make a mod for an old Minecraft version) some may not be. If you plan on modding for older versions of the game, keep in mind what version of the JDK they need and whether or not yours is compatible.

For these workshops, we will be modding in version 1.18.1, the most recent as of writing this. As such, we will use Java SE Development Kit 17.0.1, the most recent addition.

Installation

  • Go to this link: https://www.oracle.com/java/technologies/javase/jdk17-archive-downloads.html (If outdated, you may need to look this up)
  • Find the version that is right for your OS and download. You can manually install the JDK if you’d like, but I recommend downloading the installer. For Windows, that is the .exe file, and for Mac, it is the DMG
  • The installation should be rather straightforward. Follow the instructions of the installer and let it do its thing
  • After a successful installation, we’ll need to do something a bit risky to make sure your computer is pointing to the right place. You may have an older version of the JDK from programming classes or that was first installed with your computer, so we need to check the JAVA_HOME system variable so it knows where to find JDK 17
    • Info: the JAVA_HOME variable is the path your computer uses to find your JDK installation. Essentially, you’re just pointing it in the right direction. It may be hard-coded to look at a specific JDK you installed, so we’ll change it to look at JDK 17

    Windows: In your system search bar, look up ‘Environment Variables’ and click on the first system result. This will pull up a menu called System Properties. Go to the Advanced tab and click on Environment Variables at the bottom

    1. Go to system variables (the second box) and check to see if there’s a JAVA_HOME variable. If not, click New the box. If it is there, you may need to still edit the path. The name of the variable will be JAVA_HOME and the path is the installation path for JDK 17. Mine is: C:\Program Files\Java\jdk-17.0.1 Though yours may be a bit different. Be VERY sure that the path you put in is exactly where your installation is. Hint: if you go to the folder JDK 17 is in and right-click on the path bar, you can ‘Copy address as text’ and paste it in for 100% accuracy.
    1. Go to system variables (the second box) and find the Path variable. Double-click on it to edit. If you did not have to create the JAVA_HOME variable, check to see if JAVA_HOME appears in any of the system variables. If not, you’ll need to add it. On the right side, click ‘New’ and type %JAVA_HOME%\bin and then use the Move Up button on the right to move it all the way to the top of the list so the JDK loads faster when you start your computer.
    1. To check if the path worked, open Command Prompt and type (without quotations) ”echo %JAVA_HOME%” and it should print the path to JDK 17

Mac: Admittedly, I am far less familiar with how to set up the JDK on a Mac. I found a tutorial that seems to explain it far better than I could, but if there is trouble or concern you can always reach out and we can help you as best as we can. I can also reach out to people more familiar with Macs than me and redirect them to you. http://www.sajeconsultants.com/how-to-set-java_home-on-mac-os-x/ With all of this done, your JDK should hopefully be ready to go! This was the longest and most cautious part, so from here it should be much easier.

IntelliJ

IntelliJ is an Integrated Development Environment (IDE) for Java. For those of you who took Programming I and II and worked with Eclipse, that too is an IDE. It simply means that we have created a workspace for our code so that we can more easily work with it. IntelliJ offers auto-complete suggestions for code, real-time corrections to code, and a dark mode that looks very nice. Eclipse works as well, but in my opinion, IntelliJ works better and looks cleaner. For this workshop, I will be working in IntelliJ and will base a lot of my IDE instructions on IntelliJ, but if you are more comfortable with another like Eclipse or Visual Studio Code, I can assist there as well.

Installation

  1. Go to this link: https://www.jetbrains.com/idea/download/#section=windows
  1. Select your OS and download the Community version (the free one)
  1. Follow its instructions to install IntelliJ
  1. You should be ready to go, but there’s a couple of extensions we can download to make modding nicer. Pull up IntelliJ and on the right side click on Plugins
    1. In the Marketplace tab, search up Kotlin and install the first result (should just be called Kotlin)
    1. Next, look up Minecraft and install Minecraft Development (this will add auto-complete and importing support for Minecraft-related files and code)

Your IntelliJ is ready to go! Now we’ll need is a fabric project to start working in, and so we’ll need a way to open them.

ZIP Files & WinRAR

Most projects you will download directly will come in ZIP folders, compressed folders that save space and create added security (when wanted). The template we’ll download after this is in a ZIP folder, so we need a way to extract it. If you’re familiar and comfortable with Git and Github, you can extract it through that. However, I also want to cover the manual way as well. Your computer may already have a way to do this, such as WinZip, 7-Zip, jZip, and WinRAR (Mac’s come with an application for this, I believe). If you don’t have a way to open zip files, you can download WinRAR for “free.” Technically, it’s only a 30-day trial before you have to purchase a license, but so long as you keep closing their reminders that your trial is expired, you can keep using it. The full version includes some neat features, so if you feel like supporting the developers, you can purchase it if you’d like. https://www.win-rar.com/download.html?&L=0 From here, download the most recent version of WinRAR for your OS and install it. With that, you can now work with ZIP folders, and finally, we can load in the project.

Fabric & Gradle

Using the original Source Code:

Fabric is a toolkit that uses the JDK as a foundation to create more intricate tools specifically for Minecraft mods. It does a LOT of legwork behind the scenes to make modding as easy as possible for us. It also loads in the source code for Minecraft and allows us to work with it directly in our environment of choice (more on that later).

This combines with the Gradle wrapper, which is an extra specialized toolkit for version control and publishing. Fabric uses Gradle so that any mod made with Fabric is in the Gradle system, and thus can be ported in and used as a library by another mod. This makes it WAY easier to import libraries and create dependencies.

For these workshops, we have put together a clean template for us to use throughout the semester. You can find it here:

You’ll be taken to page on Github with a lot of information on the project’s history, reported issues, push information, pull requests, and lots of other cool Git stuff that we can discuss and learn later as we go through the project and learn some helpful Git techniques.

You have two options to bring the template to your computer: using Git, or manually downloading and extracting it. Either way, I recommend making a new folder on your Desktop where you can put all of our workshop materials.

Git HTTPS:

This requires GitBash or Git SCM to be installed on your computer. If you don’t have these, don’t worry about it, you can do the manual way instead. These workshops are not meant to teach you Git. Go to the top and make sure the left tab is selected to 1.18, the version of the project we want. Next, go to the right side and clock on Code, then select the small tab ‘HTTPS’. You’ll see a URL. On the right side of that URL is a little button to copy it. Click that. Next, open up your command prompt or other system console and ‘cd’ into your workshop folder. You start at your User profile on your computer. For me, my folder is on my desktop and is called Minecraft Workshop Materials, so I’ll put it in

cd Desktop\Minecraft Workshop Materials

Once there, you can type

git clone https://github.com/catanobro10/MC-Fabric-Workshop.git

and it will download a clone of the project for you.

Manually:

Go to the top and make sure the left tab is selected to 1.18, the version of the project we want. Next, go to the right side and click on Code, then click on Download ZIP. Download the project to a place you can easily find it. Finally, open it up and extract the folder into your workshop folder

  • Advanced Technique: This repository is also a template, which means you can use it to create your own repository formatted after ours that you own and have control over. This can be very nice if you decide to start modding on your own time outside of these workshops, as you can quickly create your own mod repositories for people to visit, download your mod, report bugs and issues, and have other collaborators easily merge changes.
  • All you need to do is be signed into GitHub and click “Use This Template”, give your new repository a name, make sure you’re the owner, and save. You can then use the same techniques above to clone the project

We actually don’t need to install anything else, as they come with the example project. However, if you have a version of Minecraft on your computer and want to either test your mod with the real version or just want to start exploring Fabric mods, you can go here to install Fabric for your game: https://fabricmc.net/use/installer/

Loading & Running the Project

Now we can load the project into IntelliJ and ensure that it will run properly. This may require a lot of troubleshooting and trial and error, but once you do it once, it’ll be MUCH easier every time after.

  1. Open up IntelliJ and select Open. Find the folder of the example mod (base-named fabric-example-mod-1.18). Be sure to select the extracted folder and not the ZIP folder. It may ask if you trust the project since Gradle can have some security issues related to loading foreign projects. In this case, we can trust this project, but do be careful about other people’s source code
  1. Gradle will probably start loading up immediately after opening the project. Let it do its thing in the meantime. This can take a couple of minutes to 10 minutes depending on your internet, your computer’s speed, Gradle’s server traffic at the moment, and lots of other stuff. You may also get an error message saying that JDK 17.0.1 is incompatible with a certain Gradle version. The build should sort this out automatically.
  1. IntelliJ will probably also pull up the Build console while Gradle is running and setting stuff up. If it does not, go to the bottom and click ‘Build’ to pull it up. After Gradle has done its thing, you will hopefully see BUILD SUCCESSFUL at the end of the runtime. However, a lot of errors can occur, and in that case, you can go the section right after this to debug these
  1. With a successful build, we can do a test run of the project. On the right side, you should see a small tab called Gradle. Click on that, then open up Tasks, and it will pull up lots of Gradle options. Open up fabric and double-click on runClient. Gradle will get to work building and running your project with a debug version of Minecraft. Hopefully it will simply load up after a bit of time and an instance of Minecraft will run on your computer. Check the Build console to see all the work it’s doing and if there are any errors. You will also very likely get a 401 error, which is Minecraft saying that there is no valid player on the game, so it will give you a dummy player named something like Player4836 or some other random number.

You should now have a fully set up project and environment to do some Minecraft modding! From here, you can begin to play with code yourself and take a look at the Fabric Wiki to see what sort of things you can do. In the first workshop, we will start simple by making a couple of basic items that use crafting recipes and a furnace to obtain. Therefore, you can read ahead and look at the sections on making a basic item and creating recipes.

Common Issues

More often than not, the initial Gradle build fails. Fortunately, I’ve dealt with these often enough that I can debug the most common ones quickly.

Wrong Java Bytecode: First we can check to make sure Java loaded in properly. At the top left, open File and go to Settings. In the search bar, look up ‘Java Compiler’. You should be sent to tab called exactly that. Under the “Use ‘—release’ option for cross-compilation” checkbox, click on the dropdown box and select 17 (as in JDK 17’s bytecode). Hit Apply in the bottom-right to save.

Incorrect Gradle JDK: Gradle may just be looking at the wrong JDK. If you have an older JDK installed on your computer, IntelliJ may point to that one by default, messing with Gradle. We can change this easily. In the Settings search bar, look up ‘Gradle’ and it will take you to the Gradle settings page. At the very bottom, select the dropdown for Gradle JVM and select either JAVA_HOME or JDK 17.0.1 directly. Either should work. Hit Apply in the bottom-right to save.

Missing JDK: IntelliJ may have also missed your JDK 17.0.1 installation, so you’ll have to manually add it. After you do this once, IntelliJ should save it for every project after. If JDK 17.0.1 is not an option, check to see if JAVA_HOME points to it instead. That should still work. If not, you may not have set up your JAVA_HOME variable correctly. You can go back and double-check that, and you can manually add JDK 17.0.1 to IntelliJ. Click “Add JDK...” at the bottom, which will give you a path popout. Locate your JDK 17.0.1 installation (usually in something like Program Files/Java/jdk-17.0.1) and highlight it, then click OK. Now you should be able to select JDK 17.0.1 for your project. Hit Apply in the bottom-right to save.

After checking these common errors, you should see tiny symbol of an elephant (Gradle’s logo) with a reload button towards the top-right. Click on that to reload IntelliJ and apply the changes you’ve made in the settings.

You should hopefully get a successful build after this. If not, you can reach out before the first workshop or search up your error and you may find a solution online.

Helpful Tip - Applying changes without restarting Minecraft

  • Reload changed classes In Eclipse or Intellij IDEA, run Minecraft in debug mode. To apply changes in code, click the build button in Intellij IDEA or save in Eclipse. Note: this only allows you to change method bodies. If you do any other kind of change, You will have to restart. However, if you use DCEVM (Not supported by Java 11+), you will be able to do most changes, including adding and removing methods and classes.
  • Reload assets After you make changes to assets such as textures and block/item models, you can rebuild the project and press F3 + T to apply changes without restarting Minecraft. More specifically, this is how to reload anything the mod provides as a resource pack.
  • Reload data You can apply any changes made in the data/ directory such as recipes, loot tables and tags by rebuilding the project and then using the in-game command /reload. More specifically, this reloads anything the mod provides as a data pack.
